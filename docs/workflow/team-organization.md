# Organização da Equipe

## Objetivo

Este documento apresenta como a equipe se organizou para dar continuidade ao projeto Gestão RCC durante a avaliação prática do primeiro bimestre.

O objetivo deste bimestre não é finalizar o sistema, mas organizar o repositório, criar documentação mínima, planejar a evolução do projeto e comprovar o uso de Git/GitHub em equipe.

# Organização das entregas

A equipe organizou a entrega do primeiro bimestre em quatro frentes principais:

## 1. Organização do repositório

Criação da estrutura exigida na avaliação, com pastas específicas para planejamento, fluxo de trabalho e evidências.

## 2. Documentação do projeto

Criação de documentos explicando a visão do sistema, backlog, roadmap, separação entre desenvolvimento e produção e organização da equipe.

## 3. Evidências de Git/GitHub

Registro de prints demonstrando o uso do Git, commits, push, pull e participação de mais de um colaborador.

## 4. Artefatos de acompanhamento

Criação de Kanban e Gantt em Markdown, além de versões físicas feitas em papel e fotografadas para compor as evidências da entrega.

---

# Ciclos de trabalho

A equipe adotará ciclos curtos inspirados em Scrum, chamados de sprints acadêmicas.

Cada sprint terá:

1. Definição das tarefas.
2. Distribuição entre os integrantes.
3. Execução das atividades.
4. Registro dos commits.
5. Atualização da documentação.
6. Revisão do que foi concluído.
7. Ajuste do backlog para o próximo ciclo.

# Sprint do primeiro bimestre

## Objetivo da sprint

Organizar o projeto para atender aos critérios da avaliação prática do primeiro bimestre.

## Tarefas da sprint

- Atualizar README.
- Criar visão do projeto.
- Criar backlog.
- Criar roadmap.
- Documentar separação entre desenvolvimento e produção.
- Documentar organização da equipe.
- Criar Kanban.
- Criar Gantt.

- Registrar evidências de Git/GitHub.
- Compactar o repositório com a pasta `.git`.

# Forma de acompanhamento

O acompanhamento será feito por meio de:

- arquivos Markdown dentro do repositório;
- Kanban físico;
- Gantt físico;
- prints de Git/GitHub;
- histórico de commits.

Essa escolha foi feita porque o projeto ainda está em fase inicial de organização acadêmica, e os arquivos no próprio repositório facilitam a consulta pelo professor e pela equipe.
