# Visão do Projeto

## Nome do projeto

Gestão RCC

## Visão geral

O Gestão RCC é um sistema web voltado para apoiar a organização de informações de pacientes atendidos pela Rede de Combate ao Câncer.

O projeto já possui uma base inicial desenvolvida em Flask e será continuado nesta disciplina com foco em organização, melhoria técnica, documentação, fluxo de trabalho e preparação de uma versão 1.0.

## Problema identificado

A instituição precisa lidar com informações de pacientes, registros de atendimento, serviços prestados e possíveis benefícios entregues, como cestas básicas.

Sem um sistema organizado, essas informações podem ficar espalhadas em planilhas, anotações ou controles manuais, dificultando a consulta, a atualização e o acompanhamento dos atendimentos.

## Objetivo geral

Desenvolver e organizar um sistema web que centralize informações essenciais dos pacientes e auxilie no acompanhamento dos serviços realizados pela Rede de Combate ao Câncer.

## Objetivos específicos

- Manter cadastro organizado de pacientes.
- Permitir consulta rápida dos dados cadastrados.
- Registrar serviços prestados a cada paciente.
- Apoiar o controle de entrega de benefícios, como cesta básica.
- Melhorar a organização interna do projeto.
- Criar documentação técnica e de planejamento.
- Utilizar Git e GitHub de forma adequada durante o desenvolvimento.

## Público-alvo

O sistema é voltado para:

- voluntários da Rede de Combate ao Câncer;
- responsáveis pelo cadastro de pacientes;
- equipe administrativa;
- pessoas responsáveis pelo acompanhamento de serviços e benefícios;
- integrantes da equipe de desenvolvimento do projeto.

## Funcionalidades principais previstas

### Pacientes

- Cadastro de pacientes.
- Listagem de pacientes.
- Edição de dados cadastrais.
- Exclusão controlada de pacientes.
- Visualização dos detalhes do paciente.

### Serviços

- Registro de serviços prestados.
- Histórico de serviços por paciente.
- Consulta de atendimentos anteriores.

### Cesta básica

- Registro da última entrega.
- Controle de frequência de entrega.
- Identificação de pacientes com cesta atrasada.
- Atualização da entrega pelo sistema.

### Usuários e segurança

- Login de usuários.
- Proteção de rotas.
- Separação futura de permissões.
- Remoção de dados sensíveis do repositório.

### Interface

- Padronização visual das telas.
- Melhoria na navegação.
- Melhor organização das informações.
- Criação futura de uma tela inicial ou dashboard.

## Escopo da versão 1.0

A versão 1.0 deverá conter uma entrega funcional e organizada, incluindo:

- cadastro e listagem de pacientes;
- edição de pacientes;
- tela de detalhes;
- registro básico de serviços;
- controle básico de cesta básica;
- login funcional;
- documentação atualizada;
- repositório organizado e versionado.

## Fora do escopo da versão 1.0

Algumas ideias poderão ser consideradas futuramente, mas não fazem parte da entrega principal da avaliação final:

- aplicativo mobile;
- integração com WhatsApp;
- relatórios avançados;
- dashboard completo com gráficos;
- controle financeiro;
- automações externas complexas.
